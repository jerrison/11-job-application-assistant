# Execution Plans

Execution plans are the repo's living record for complex, multi-step work.

- Put in-flight plans in `docs/exec-plans/active/`.
- Move finished plans to `docs/exec-plans/completed/`.
- Start from `docs/PLAN_TEMPLATE.md`.
- Keep the `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` sections current while the work is active.
- Keep milestones self-contained and independently verifiable so another agent can resume the work from the repo alone.
